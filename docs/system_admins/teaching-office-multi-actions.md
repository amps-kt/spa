# Teaching Office Student Submission Management

<!-- This is a prety complicated page, so here's some notes on how the page works that can later become proper documentation.-->

The page is split into 2-4 sections, depending on how you want to count things. 

0. Flag Selection
Right under the usual page title users can switch tabs to view, and adjust the submission details for, a particular group of students. 

1. Apply to
Next section allows users to make a mass-selection, they must pick at least one Unit of Assessment and then can choose to eiter include or exclude students from all the available students with this flag. This selection can be seen on the data table below, and unless a valid selection exists, the user cannot actually trigger any of the actions in the section below.

2. Quick Actions
There are two:
  - you can change the submission status for the selected units and students
  - you can change the due date for the selected units and students

3. The Data Table
This is where you can make all the granular changes you might need by interacting with the data table directly. The cells in the "Weight", "Due Date", "Submitted?" and "Enrolled?" columns are interactive, you just click on them and make the change you want. These changes are not immediately applied so don't worry if you misclick - dangerous operations are heavily guarded.

Once you've made the changes you want, either via tha "Quick Actions" or by interacting with the data table directly you can choose to either discard them or review and save them. If you choose to save these changed you are shown a summary and given the optino to view all the changes you are about to commit more granularly. You can make changes in multiple different tabs at the same time and not worry about whether they are discarding or saving changes from other tabs, everything is scoped to the tab you have open. Once you are sure that you want to commit these changes, just press the "Commit Changes" button and you're done!
